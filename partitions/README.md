# Non-IID Partition Files

This directory holds the JSON partition files generated by the Dirichlet partition
generator `partitions/generate.py`. Each file describes how the WHU-GCD training
set is split across federated clients using a Dirichlet distribution that controls
the heterogeneity between clients.

**These JSON files are gitignored** (each ~8 MB — regeneratable on demand).

## Partition Method: Dirichlet

The Dirichlet concentration parameter `alpha` controls how heterogeneous the
clients are. For each semantic class, a Dirichlet(`alpha`, …, `alpha`) sample
gives the per-client proportions; a smaller `alpha` → more skewed (more
Non-IID), `alpha` → ∞ approaches IID (uniform).

| alpha | Heterogeneity |
|---|---|
| 0.1 | 极端异构 (extreme) |
| 0.5 | 中等异构 (moderate) |
| 1.0 | 轻度异构 (mild) |
| 100.0 | 接近 IID (near-IID) |

**Class 0 (no-change) is treated as an independent semantic class**, on equal
footing with the 6 change classes {2..7}. Negative samples
(ucd / ugcd / ugcd_full) are labeled class 0 and participate in the Dirichlet
draw alongside the gcd change classes — so each client gets a skewed mix of both
change classes and negatives.

## File Naming Convention

| File | Clients | alpha | Heterogeneity |
|---|---|---|---|
| `partition_dirichlet_a0.1_n7.json` | 7 | 0.1 | Statistical (extreme) |
| `partition_dirichlet_a0.5_n7.json` | 7 | 0.5 | Statistical (moderate) |
| `partition_dirichlet_a1.0_n7.json` | 7 | 1.0 | Statistical (mild) |
| `partition_dirichlet_a100.0_n7.json` | 7 | 100.0 | Near-IID |

Naming pattern: `partition_dirichlet_a{alpha}_n{num_clients}.json`.

## Regenerating Partitions

From the project root (`FedChange/`), with `../WHU-GCD/` containing the dataset:

```bash
# The 4 canonical partitions used by the run scripts
python -m partitions.generate --alpha 0.1  --num_clients 7 --data_root ../WHU-GCD
python -m partitions.generate --alpha 0.5  --num_clients 7 --data_root ../WHU-GCD
python -m partitions.generate --alpha 1.0  --num_clients 7 --data_root ../WHU-GCD
python -m partitions.generate --alpha 100  --num_clients 7 --data_root ../WHU-GCD

# Exclude negative samples (ucd/ugcd/ugcd_full) from the partition
python -m partitions.generate --alpha 0.5 --num_clients 7 --no_neg

# Preview the split statistics without writing the file
python -m partitions.generate --alpha 0.5 --dry_run
```

## JSON Schema

```jsonc
{
  "partition_method": "dirichlet",   // always "dirichlet"
  "num_clients": 7,
  "params": {
    "alpha": 0.5,
    "num_clients": 7,
    "include_neg": true
  },
  "clients": {
    "client_0": {
      "samples": [
        {
          "im1":   "train/gcd/im1/x.png",       // relative to data_root
          "im2":   "train/gcd/im2/x_5.png",
          "label": "train/gcd/label/x_5.png",
          "mask1": "train/gcd/mask1/x.png",
          "mask2": "train/gcd/mask2/x_5.png",
          "source": "gcd"
        }
        // ...
      ],
      "stats": {
        "total": 540,
        "sources": { "gcd": 420, "ucd": 80, "ugcd": 40 },
        "class_dist": { "0": 120, "5": 320, "6": 100 }
      }
    }
    // ...
  }
}
```

All paths are stored **relative to `data_root`**, so a partition JSON works on
any machine — just point `--data_root` at the local WHU-GCD directory.
